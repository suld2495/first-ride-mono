**1. 현재 상황**

**1-1. 서비스의 원래 지향점**

우리 앱이 목표로 하는 것은 단순히 루틴을 기록하는 도구가 아니다. 실제
게임 테마를 앱 전체에 입혀, 사용자가 \'앱을 플레이하기 위해 루틴을
수행하는\' 경험을 만드는 것이다. 기존 루틴 앱들이 사용자 스스로의 의지에
의존하는 구조라면, 우리는 게임적 서사와 보상 체계가 그 동기를 대신하는
구조를 지향한다.

**1-2. 현재 앱의 실제 구조**

그러나 현재 구현된 앱은 아래와 같은 특성을 띠고 있다.

---

**구분** **현황**

**루틴 유형** 목표형 루틴 중심 --- 메이트 지정, 인증 사진 등
외부 검증 구조가 기본

**습관 형성 지원** 반복·지속을 돕는 구조 미흡. 연속 기록(스트릭),
대체 루틴 등 부재

**게임 테마 연동** 테마와 루틴 달성이 독립적으로 존재. 테마가
동기부여로 작동하지 않음

**동기부여 구조** 현재 요소: PB(포인트·배지) 수준. 사용자 스스로
동기를 만들어야 하는 구조

**2. 타겟 정의**

서비스 방향을 재설정하기에 앞서, \'우리가 누구를 위한 앱을 만드는가\'를
명확히 할 필요가 있다. 타겟이 흔들리면 기능 우선순위도, 게임 테마의
방향도 결정하기 어렵다.

**2-1. 핵심 타겟**

---

**타겟 정의** **설명**

**기존 루틴 앱 이탈 Streaks, 마이루틴 등을 써봤지만 동기부여 부족으로
경험자** 오래 이어가지 못한 사람. 의지보다 구조가 필요한
유형이다.

**게임을 즐기는 RPG/전략 장르를 즐기며 성장과 보상 피드백에
사람** 반응하는 유저. 루틴 앱보다 게임 앱으로 포지셔닝했을
때 더 자연스럽게 유입되는 층이다.

교집합** 동기를 유지하기 어려운 사람. 이 교집합이 우리 앱의
핵심 사용자다.

**4. 개선이 필요한 영역**

**4-1. 습관 형성 지원 기능 (현재 없음)**

- 연속 기록(스트릭): 며칠 연속으로 루틴을 수행했는지 시각적으로 확인할
  수 있어야 한다. 기록을 끊고 싶지 않다는 심리가 지속의 동력이 된다.

- 불완전한 날의 처리: 컨디션이 나쁜 날 작은 루틴이라도 수행하면
  인정받는 구조가 필요하다. 실패를 \'완전한 실패\'로 처리하면 사용자가
  이탈한다.

- 루틴 모드: 오늘 하기 어려운 루틴을 더 가벼운 대체 루틴으로 바꿀 수
  있는 옵션. 완벽주의 성향의 사용자가 앱을 이탈하는 가장 큰 이유를
  막는 장치다.

**4-2. 테마-루틴 통합 (현재 분리됨)**

현재 게임 테마는 앱의 외형에만 적용되어 있고, 루틴 달성 여부가 게임
세계에 실제 영향을 주지 않는다. 이것이 우리 앱의 핵심 차별점이 되려면 두
요소가 연결되어야 한다.

+-----------------------------------------------------------------------+
| **통합 방향 예시 (현재 없는 기능, 향후 고려 사항)** |
| |
| 판타지 마을을 배경으로, 사용자가 루틴을 달성할수록 마을이 위기에서 |
| 회복되는 구조. 루틴을 완수하지 못한 날이 쌓이면 마을에 어둠이 |
| 짙어지는 식의 서사 연동. 이렇게 되면 루틴 달성이 게임 플레이가 되고, |
| 앱을 열어야 할 이유가 생긴다. |
+-----------------------------------------------------------------------+

**4-3. 동기부여 체계 강화**

- 현재 PB(포인트·배지) 수준에서 PBL(포인트·배지·리더보드) + 레벨
  시스템으로 확장

- 출석체크형 요소: 매일 접속 자체에 작은 보상. 연속 달성일수에 따른
  보상 차등 적용

- 달성 시 보상, 미달성 시 패널티의 양방향 구조 도입 검토 --- 경쟁 앱
  분석 결과, 패널티 구조(마이루틴, Habitica)가 지속률에 유의미한
  영향을 줌 (부록 참고)

**4-4. UX 개선**

- 루틴 추가 플로우: 기본은 심플 뷰, 확장 시 세부 설정 노출. 진입
  장벽을 낮춰야 한다.

- 애니메이션: 가볍고 빠르게. 애니메이션 대기 시간이 이탈 원인이 된다.

- 인증 방식 다양화: 사진 인증 외 간단한 체크 방식도 허용. 사진 촬영
  자체가 루틴 수행의 장벽이 되어서는 안 된다.

**7. 마무리**

| 현재 우리 앱이 틀린 방향을 가고 있다는 것이 아니다. 목표한 방향으로 |
| 가기 위한 기반이 아직 완성되지 않은 상태라는 것이다. |
| |
| 게임 테마로 루틴을 즐겁게 만든다는 비전은 유효하다. 다만 그 비전이 |
| 실제로 작동하려면, 루틴 앱으로서의 기본 체력---습관 형성 지원, 지속 |
| 동기 구조---이 먼저 갖춰져야 한다. 이 문서를 바탕으로 팀 내에서 |
| 우선순위와 개발 범위를 함께 논의해주길 바란다. |
