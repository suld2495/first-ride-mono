# 모바일 로그인 화면 검증 PRD

## 1. 목적

모바일 앱의 로그인 화면에서 아이디/비밀번호 로그인 흐름이 정상적으로 동작하는지 검증한다.

## 2. 검증 대상

- 화면: 로그인 화면
- 로그인 방식: 아이디/비밀번호 로그인

## 3. 테스트 계정

- 아이디: `Fff1234`
- 비밀번호: `12341234`

## 4. 요구사항

### 요구사항 1. 빈 값으로 로그인 시도

사용자가 로그인 화면에서 아이디와 비밀번호를 모두 입력하지 않은 상태로 `로그인` 버튼을 누를 수 있다.

수용 기준:

- 아이디 입력 필드 아래에 `아이디를 입력해주세요.` 문구가 표시된다.
- 비밀번호 입력 필드 아래에 `비밀번호를 입력해주세요.` 문구가 표시된다.
- 로그인은 진행되지 않는다.
- 화면 이동은 발생하지 않는다.

### 요구사항 2. 잘못된 아이디 또는 비밀번호 입력

사용자가 로그인 화면에서 아이디 또는 비밀번호를 잘못 입력한 상태로 `로그인` 버튼을 누를 수 있다.

수용 기준:

- 서버 로그인 요청이 수행된다.
- 로그인 성공 처리는 되지 않는다.
- 사용자에게 `아이디 또는 비밀번호가 일치하지 않습니다.` 문구가 표시된다.
- 로그인 이후 화면으로 이동하지 않는다.

검증 예시:

- 아이디: `Fff1234`
- 비밀번호: `wrongpassword`

또는

- 아이디: `wrongid`
- 비밀번호: `12341234`

### 요구사항 3. 올바른 아이디와 비밀번호 입력

사용자가 로그인 화면에서 올바른 아이디와 비밀번호를 입력한 뒤 `로그인` 버튼을 누를 수 있다.

수용 기준:

- 아이디 `Fff1234`와 비밀번호 `12341234`를 입력하면 로그인에 성공한다.
- 로그인 성공 후 루틴 화면 `/(tabs)/(afterLogin)/(routine)` 으로 이동한다.
- 로그인 실패 메시지는 표시되지 않는다.

## 5. 검증 판정 기준

- 위 3개 요구사항이 모두 충족되면 `pass`로 판정한다.
- 하나라도 충족되지 않으면 실패한 요구사항별로 실제 동작과 재현 절차를 기록하고 `fail` 또는 `partial`로 판정한다.
