# 모바일 앱 요구사항 검증

Maestro CLI로 React Native 앱을 직접 조작하며 요구사항이 실제로 충족되는지 확인한다.

**핵심 원칙:** 코드가 아닌 동작을 검증한다. e2e 테스트를 작성하는 것이 아니라 요구사항 충족 여부를 확인하는 것이다. 검증용 flow는 일회성이다.

## 언제 사용하는가

- React Native 화면/기능 구현 후 요구사항 충족 여부 확인
- 화면 전환, 입력, 버튼 동작 등 사용자 흐름 검증
- 에러 상태 및 경계 조건이 앱에서 올바르게 표시되는지 확인

## 사전 조건

- 에뮬레이터 또는 실기기가 연결되어 있어야 한다
- 앱이 빌드되어 설치된 상태여야 한다
- `maestro` CLI가 설치되어 있어야 한다 (`brew install maestro`)

## 요청 방법

**1. verify 서브에이전트 실행:**

Task 도구를 사용해 `mobile-verifier.md` 템플릿을 verify 서브에이전트에게 전달한다.

**플레이스홀더:**
- `{REQUIREMENTS}` — 검증할 요구사항 (화면 흐름 또는 기능 명세)
- `{APP_PACKAGE}` — 앱 패키지명 (Android: com.example.app / iOS: com.example.App)
- `{DESCRIPTION}` — 무엇을 검증하는지 간단한 요약

**2. 결과 반영:**
- 실패 항목은 즉시 수정 후 재검증
- 통과 항목은 완료로 기록

## 예시

```
[로그인 화면 구현 완료]

[verify-mobile 서브에이전트 실행]
  REQUIREMENTS: 사용자는 이메일과 비밀번호를 입력해 로그인할 수 있다.
                로그인 성공 시 홈 화면으로 이동한다.
                실패 시 화면에 오류 문구가 표시된다.
  APP_PACKAGE: com.myapp.android
  DESCRIPTION: 로그인 화면 흐름 검증

[서브에이전트 결과]:
  통과: 올바른 자격증명으로 로그인 후 홈 화면 이동 확인
  통과: 잘못된 비밀번호 입력 시 오류 문구 표시 확인
  실패: 비밀번호 입력 중 키보드가 화면을 가림 — 레이아웃 문제
  판정: 일부 실패

[수정 후 재검증]
```
